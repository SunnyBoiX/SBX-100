I originally had a vision of creating a huge keyboard with OLED, a rotary encoder, and LEDs, but I realized this was going to be one struggle during the process of my schematic. I originally had a numpad, but I had to remove it because the Raspberry Pi Pico does not have enough GPIO pins, and thus, I wanted to make the keyboard wireless in order to fulfill my happiness. However, this was a terrible idea as well because microcontrollers that are wireless, such as nice!nano or nrf types, are really expensive and can also be hard to route during the PCB making. I finally made my decision on keeping this project as my learning experience, so if I manage to complete it successfully, I can at least learn the fundamentals on how to make a keyboard, which I can create a new version next time with all the cool stuff I wished for and had to leave out. This keyboard, it is a 100-switch keyboard that also has a rotary encoder switch, and its microcontroller is the Raspberry Pi Pico. I honestly had no idea how to start this project after finding out that a matrix is required for this, so I created one in a different sheet. The main sheet just has the Raspberry Pi Pico and four mounting holes. I connected all the GPIO pins with the global labels I used from the matrix. I disconnected the GPIO pins I didn't need, and still have one GPIO pin left. I'm thinking of adding LEDs, which I might do now. Almost completing this schematic took me over a few days because of how busy I got with events, as it is summer. Overall, my end goal now is to create a 100-switch wired keyboard that has a rotary encoder and LEDs inside of it and is, of course, wired. All I have left to do for the schematic is LEDS now!

I stored all the LED data in a single global label, and instead of using wires for the next rows, I decided to use Net Labels to make it possible. I completely forgot one crucial step, which is to assign footprints, which I will do now.
